## 2. Coolify 실무 배포 매뉴얼

직원분께서 Coolify 콘솔에 접속하여 본 웹앱을 호스팅 배포할 때 아래 순서대로 입력 및 세팅을 진행해 주시면 됩니다.

### 1단계: 깃허브 업로드 및 새 리소스 로드
1.  작업이 완료된 소스코드를 연결된 깃허브 저장소(예: `naver-place-search`)의 `main` 브라우저에 **Push(업로드)**합니다.
2.  Coolify 대시보드 ➔ **`Projects`** ➔ **`production`** 환경으로 진입합니다.
3.  우측 상단의 **`+ New`** (또는 `+ New Resource`) ➔ **`Private Repository (with GitHub App)`**를 누릅니다.
4.  내 저장소 목록 중 업로드한 저장소를 선택하고 **`Load Repository`**를 클릭합니다.
5.  **Configuration 설정**:
    *   **Build Pack**: **`Dockerfile`** (Nixpacks 대신 Dockerfile로 자동 지정됩니다.)
    *   **Port**: **`5000`** (Flask 앱 기본 바인딩 포트)
6.  맨 아래 **`Continue`** 버튼을 클릭합니다.

### 2단계: 도메인 매칭 (🚨 무한 리디렉션 방지)
1.  생성된 애플리케이션의 **General 설정 화면**으로 진입합니다.
2.  화면 중간의 **`Domains`** 입력 칸에 접속 주소를 입력합니다.
3.  **[주의]** Cloudflare Full Strict 모드와의 무한 핑퐁 리디렉션을 막기 위해 프로토콜은 **반드시 `http://`로 지정**합니다.
    *   *서브도메인 입력 예시:* **`http://place-search.yourdomain.com`**
    *   *한글 도메인 입력 예시 (퓨니코드 필수):* **`http://xn--...`**
4.  도메인 입력 후 바로 위의 **`Save`** 버튼을 누릅니다.

### 3단계: 환경 변수(Environment Variables) 입력
좌측 사이드 메뉴의 **Environment Variables** 탭으로 이동하여 다음 변수들을 등록합니다.

| 환경 변수 Key | Value 설정 값 | 비고 |
| :--- | :--- | :--- |
| **`PLAYWRIGHT_HEADLESS`** | **`True`** | 브라우저를 화면 없이 백그라운드로 작동시킴 (필수) |
| **`DATA_DIR`** | **`/app/data`** | 데이터 저장 위치를 영구 볼륨 공간으로 매핑 (필수) |
| **`NAVER_SEARCHAD_API_KEY`** | *(실제 검색광고 API KEY)* | 미입력 시 소스코드 내 기존 로컬 키로 fallback 작동 |
| **`NAVER_SEARCHAD_SECRET_KEY`** | *(실제 검색광고 SECRET KEY)* | 미입력 시 소스코드 내 기존 로컬 키로 fallback 작동 |
| **`NAVER_SEARCHAD_CUSTOMER_ID`** | *(실제 검색광고 CUSTOMER ID)* | 미입력 시 소스코드 내 기존 로컬 키로 fallback 작동 |

### 4단계: 영구 볼륨 마운트 설정 (🚨 데이터 유실 방지)
서버가 재시작되어도 검색 이력이 보존되도록 볼륨 마운트를 지정합니다.

1.  좌측 사이드 메뉴의 **Storages** (또는 `Storage`) 탭으로 이동합니다.
2.  **`+ Add Volume`** 또는 신규 볼륨 추가 버튼을 클릭합니다.
3.  설정 입력창에 다음과 같이 바인딩합니다.
    *   **Destination Path (컨테이너 내부 경로)**: **`/app/data`**
    *   **Volume Name / Source Path**: *(지정하고 싶은 이름 예: `naver-place-data` 입력)*
4.  저장하여 설정을 완료합니다.

### 5단계: 배포 개시 및 최종 검증
1.  우측 상단의 보라색 **`Deploy`** 버튼을 누릅니다.
2.  도커 빌드가 정상적으로 완료되어 상태 표시가 **`Running`**으로 바뀔 때까지 기다립니다.
3.  상태가 가동 중으로 전환되면 브라우저에 설정한 도메인 주소(예: `https://place-search.yourdomain.com` - 접속 시에는 브라우저가 https로 리디렉션 처리함)를 치고 접속하여 정상 작동하는지 테스트합니다.
