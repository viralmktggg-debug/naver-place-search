# Coolify - 신규 프로젝트(Private Repo) 호스팅 배포 매뉴얼

이 문서는 벌쳐(Vultr) VPS + Coolify 환경에서 깃허브의 비공개(Private) 저장소를 연동하고, 새로운 웹/앱 애플리케이션을 최초로 호스팅하는 전체 과정을 순차적으로 기록한 표준 가이드입니다.

---

## 1단계: Coolify 상세 페이지 탈출 및 새 리소스 생성 진입
새로운 서비스를 추가하려면 기존 서비스(예: 전자계약) 상세 설정창에서 상위 프로젝트 경로로 이동해야 합니다.

1. **Coolify 콘솔 대시보드**에 접속합니다.
2. 화면 최상단 좌측의 빵부스러기(Breadcrumb) 경로인 `My first project > production > 전자계약` 중에서 **`production`** 글자를 클릭합니다.
3. 환경(Environment: production) 메인 화면으로 빠져나오면, 우측 상단에 나타나는 **`+ New`** 또는 **`+ New Resource`** 버튼을 클릭합니다.

---

## 2단계: 깃허브 프라이빗 앱(GitHub App) 최초 연동 (연동 다리 건설)
소스코드가 비공개(Private) 상태이므로 서버가 깃허브에 접근할 수 있도록 보안 통로를 뚫어주는 과정입니다. (계정당 최초 1회만 수행, 이후에는 이 단계 패스 가능)

1. New Resource 메인 화면의 'Applications > Git Based' 영역에서 두 번째에 있는 **`Private Repository (with GitHub App)`** 버튼을 클릭합니다.
2. `No GitHub Application found` 팝업창이 뜨면 **`Name`** 칸에 관리용 이름(예: `gonggam-github`)을 적고 아래 **`Continue`** 버튼을 클릭합니다.
3. 화면이 바뀌면 왼쪽 **Automated Installation** 박스 맨 아래에 있는 보라색 **`Register Now`** 버튼을 클릭합니다.
4. 브라우저에 새로운 깃허브(GitHub) 창이 열리면 로그인을 진행한 후, 권한 승인 화면에서 다음을 세팅합니다.
   * **Repository access:** **`All repositories`**를 선택하거나, `Only select repositories`를 선택한 후 이번에 배포할 저장소(예: `allin1`)를 명시적으로 체크합니다.
5. 맨 아래 초록색 **`Install & Authorize`** (또는 **`Save`**) 버튼을 클릭하면, 권한 승인이 완료되면서 Coolify 대시보드 화면으로 자동으로 복귀합니다.

---

## 3단계: 비공개 리포지토리(Repository) 선택 및 기본 검증
연동된 깃허브 다리를 통해 배포할 진짜 소스코드를 서버로 불러오는 단계입니다.

1. Coolify 좌측 사이드바 메뉴에서 **`Projects`**를 클릭합니다.
2. `My first project` ➔ **`production`** 환경을 순서대로 다시 클릭하여 진입합니다.
3. 우측 상단의 **`+ New`** 버튼을 누른 뒤, 다시 한번 **`Private Repository (with GitHub App)`** 버튼을 클릭합니다.
4. 이제 "없다"는 에러 대신 내 비공개 저장소 목록이 자동으로 팝업됩니다. 드롭다운 메뉴에서 이번에 배포할 저장소인 **`allin1`**을 선택하고 **`Load Repository`**를 누릅니다.
5. 아래 **Configuration** 기본 설정값들을 확인합니다.
   * **Branch:** `main` (또는 실제 코드가 포함된 브랜치 명)
   * **Build Pack:** `Nixpacks` (Coolify 자동 빌드 엔진)
   * **Port:** **`3000`** (기존 전자계약 서비스와 포트 번호가 `3000`으로 완전히 같아도 컨테이너가 분리되어 있어 충돌하지 않으므로 그대로 유지)
6. 검증이 끝났다면 맨 아래 **`Continue`** 버튼을 클릭합니다.

---

## 4단계: 도메인 매칭 및 최종 배포(Deploy) 개시
서버 내부에 독립된 방(Container)이 개설되었으므로 외부 사용자가 접속할 도메인 주소를 명시하고 가동하는 단계입니다.

1. `Continue`를 누르면 새로 만들어진 애플리케이션의 **General 설정 화면**으로 진입합니다.
2. 화면 중간의 **`Domains`** 입력 칸을 찾습니다.
3. 올인원 전용으로 준비한 서브도메인의 **한글 퓨니코드 주소**를 프로토콜을 포함하여 정확히 적어줍니다.
   * *올바른 입력 예시:* `https://xn--...` (http가 아닌 **https://** 로 시작 필수)
4. 도메인 입력 후 바로 위에 있는 **`Save`** 버튼을 눌러 설정을 저장합니다.
5. 우측 상단에 활성화된 보라색 **`Deploy`** (또는 **`Redeploy`**) 버튼을 클릭합니다.
6. **결과 확인:** 서버가 깃허브에서 코드를 받아와 자동으로 빌드를 시작합니다. 상단의 상태 표시가 **`Running`**으로 바뀌면 웹 브라우저 주소창에 설정한 한글 도메인을 치고 들어가 정상 노출되는지 최종 수동 검증합니다.
